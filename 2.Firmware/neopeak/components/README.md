组件目录（components）项目架构总览

本目录用于沉淀板级抽象、驱动与服务/应用层的模块化实现，形成清晰的分层结构，便于复用与扩展。

目录分层说明（components 下的功能目录）：

- board
	- 板级抽象层（Board Support），对硬件资源进行统一封装与配置。
	- 提供板卡初始化、引脚映射、外设复用等能力，向上屏蔽具体硬件差异。

- hal_board (HAL)
	- 硬件抽象层（Hardware Abstraction Layer），对底层外设访问进行再抽象。
	- 向上提供统一的硬件访问接口，向下对接 drivers 与底层 SDK。

- drivers
	- 设备驱动层，包含 SPI/I2C/GPIO/PWM 等外设驱动与器件驱动。
	- 负责与硬件寄存器或底层 SDK 交互，提供稳定的设备访问接口。

- services
	- 服务与应用层，包含通信状态机、算法以及应用逻辑等。
	- 通过 message bus 进行数据发布与订阅，组织业务流程。

文档规范：

每一个目录下都应包含一个 README/MD 文档，用于说明：

- 该目录的目标与职责边界
- 允许放置的模块类型与开发指导

- 典型用法与示例

本 README 用于总结整体架构与分层职责，作为 components 目录的总览说明。
